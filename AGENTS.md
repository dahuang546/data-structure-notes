## 常用操作
- 编译：`gcc -o output/xxx xxx.c`（输出到 output/，已被 gitignore）
- 推送：直接 `git push`，Token 已配置在钥匙串
- 日记：统一使用 `学习笔记.md` 单文件，按日期条目往下追加，不每天新建文件；文件顶部注明作者、目的及"本笔记和仓库主要由我的 Agent 协助管理"

## GitHub 认证方式
本仓库已配置多种 GitHub 认证方式，按需使用：

| 方式 | 命令 | 适用场景 |
|-----|------|---------|
| **Git 钥匙串**（已配置） | `git push` | 日常推送，最常用 |
| **GitHub CLI 设备流** | `gh auth login --web` | 需要 `gh` 命令行工具权限时 |
| **GitHub CLI Token** | `gh auth login --with-token < token.txt` | 脚本化或自动化场景 |

**`gh auth login --web` 使用步骤：**
1. 运行命令，复制输出的 one-time code
2. 浏览器打开 https://github.com/login/device
3. 粘贴 code 并授权

## 协作说明

> **本仓库由 AI Agent 协助管理，人类作者负责原始代码编写与注释。**
>
> 日常维护工作（如整理目录、推送代码、撰写学习日志、Git 操作等）由 AI 根据本文件设定执行。  
> 作者专注于数据结构的学习、代码实现与注释编写。  
> AI 的完整设定与操作规范见本 `AGENTS.md` 文件。

### AI Agent 功能
- 📤 **Git 推送**：代码更新后执行 `git add / commit / push`
- 📝 **学习笔记**：在单文件学习笔记中按日期条目追加日记，不新建每日文件；文件顶部不写具体日期，保留作者/目的/Agent 管理说明；学习日志用 `### YYYY年M月D日` 作为条目，同一天内有后续更新可用 `### YYYY年M月D日（续）`；同步维护英文版学习笔记（`learning-notes.md`），内容与格式保持一致
- 📁 **目录整理**：按数据结构类型（顺序表、链表、栈、队列、串等）归类代码文件
- 🔧 **配置维护**：管理 `.gitignore`、Git 凭证、GitHub CLI 配置等
- 📋 **代码辅助**：补充注释、检查代码风格、回答数据结构相关问题
- 🌐 **README 维护**：保持 README.md 中英文双语版本（中文在前，英文在后，内容一致）
- 🔄 **自我维护**：每次推送完成后，自动回顾本轮对话内容，判断是否有新的协作策略、约定或偏好需要写入/更新到本 `AGENTS.md` 文件中，主动进行维护；用户要求重启前也需执行此检查
